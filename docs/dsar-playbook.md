# Gumon DSAR Playbook (TH)

อัปเดตล่าสุด: 2026-02-13  
ขอบเขต: คำขอสิทธิข้อมูลส่วนบุคคลจากผู้ใช้เว็บไซต์ Gumon

## DSAR คืออะไร
DSAR (Data Subject Access Request) คือคำขอจากเจ้าของข้อมูล เช่น ขอเข้าถึงข้อมูล ขอแก้ไข ขอให้ลบ ขอจำกัดการใช้ข้อมูล หรือคัดค้านการประมวลผล

## บทบาททีม
1. `Data Governance Lead`
- อนุมัติผลคำขอ/กรณีซับซ้อน
- ตัดสินใจในกรณีมีข้อยกเว้นทางกฎหมาย

2. `Privacy Operations`
- รับคำขอผ่านอีเมล
- ตรวจสอบตัวตนผู้ร้องขอ
- ประสานทีมเทคนิคเพื่อค้นหา/ลบ/แก้ไขข้อมูล

3. `Security Response`
- สนับสนุนการค้น log และประเมินความเสี่ยง
- ตรวจสอบเหตุผิดปกติที่เกี่ยวกับข้อมูล

## SLA ที่แนะนำ
1. ตอบรับคำขอ (Acknowledgement): ภายใน 72 ชั่วโมง
2. ปิดคำขอทั่วไป: ภายใน 30 วัน
3. คำขอซับซ้อน: แจ้งเหตุผลและเวลาที่ต้องขยายให้ผู้ร้องทราบก่อนครบกำหนด

## Workflow ปฏิบัติการ
1. รับคำขอ
- ช่องทางหลัก: `data@gumon.io`
- สร้าง ticket: `DSAR-YYYYMM-XXXX`

2. ตรวจสอบตัวตน
- ขอข้อมูลยืนยันขั้นต่ำ: อีเมลที่ใช้ติดต่อ + รายละเอียดคำขอ
- หากยืนยันตัวตนไม่ได้ ให้หยุดกระบวนการและแจ้งเหตุผล

3. จัดประเภทคำขอ
- Access / Rectification / Erasure / Restriction / Objection / Consent Withdrawal

4. ค้นหาข้อมูลที่เกี่ยวข้อง
- Contact data
- Email correspondence
- Consent record
- Security/analytics ที่เชื่อมโยงตัวตนได้ (ถ้ามี)

5. ประเมินข้อกฎหมาย
- มีเหตุให้ปฏิเสธบางส่วนหรือไม่ (เช่น ข้อบังคับตามกฎหมาย)
- ถ้าปฏิเสธ ต้องแจ้งเหตุผลให้ชัด

6. ดำเนินการตามคำขอ
- Access: ส่งสำเนาข้อมูลตามขอบเขตที่กฎหมายอนุญาต
- Rectification: แก้ไขข้อมูลให้ถูกต้อง
- Erasure: ลบหรือทำให้ไม่สามารถระบุตัวตนได้
- Restriction/Objection: จำกัดหรือหยุดการประมวลผลตามเงื่อนไข

7. ปิดงานและบันทึกหลักฐาน
- บันทึกวันที่รับคำขอ วันที่ปิดงาน ผู้ดำเนินการ และผลลัพธ์
- เก็บบันทึกตาม Retention Matrix

## เทมเพลตตอบกลับ (สั้น)
1. รับเรื่อง
> เราได้รับคำขอของคุณแล้ว (เลขอ้างอิง: DSAR-XXXX) และกำลังตรวจสอบข้อมูลเพิ่มเติม

2. ขอข้อมูลยืนยัน
> เพื่อความปลอดภัยของข้อมูล กรุณายืนยันอีเมล/รายละเอียดเพิ่มเติมตามที่ระบุ

3. ปิดคำขอ
> เราได้ดำเนินการตามคำขอของคุณเรียบร้อยแล้ว หากต้องการข้อมูลเพิ่มเติม สามารถตอบกลับอีเมลนี้ได้

## จุดควบคุมสำคัญ
1. ห้ามส่งข้อมูลส่วนบุคคลทางช่องทางที่ไม่ปลอดภัย
2. ห้ามเปิดเผยข้อมูลของบุคคลอื่นในคำตอบ DSAR
3. ทุกเคสต้องมีหมายเลขอ้างอิงและบันทึกการตัดสินใจ
